# قواعد المشروع الخاصة

هذا الملف مشترك بين صاحب المشروع وTera.

أي قاعدة موجودة هنا تعتبر قيدًا ملزمًا للمشروع الحالي بعد اعتمادها، ويجب على Tera تمريرها للعملاء الفرعيين عند التفويض إذا كانت مرتبطة بالمهمة.

---

## 1. قواعد معتمدة

### 1.1 القواعد العامة

```text
- لا تستخدم مكتبات مدفوعة.
- لا تستخدم أي API خارجي إلا بموافقة صريحة.
- لا تخزن كلمات المرور كنص صريح.
- لا تستخدم ألوان خارج هوية الشركة.
- لا تنشئ Dashboard في النسخة الأولى.
- لا تنفذ Export Excel في النسخة الأولى.
- يجب دعم اللغة العربية RTL.
- يجب أن يكون المشروع قابلًا للتشغيل محليًا.
```

### 1.2 قواعد تتبع المهام (Traceability)

```text
- يجب أن تكون حالة كل مهمة متسقة في جميع ملفات project-control،
  خصوصًا TASK_REGISTRY.md وملف المهمة نفسه.
- لا يجوز إغلاق مهمة في السجل العام (TASK_REGISTRY.md) مع بقاء حالتها
  Draft داخل ملف المهمة (tasks/TASK-XXXX.md).
- عند تغيير حالة مهمة، يجب تحديث كلا الملفين في نفس العملية.
```

### 1.3 قواعد مراجعة القواعد قبل التنفيذ (Rules Review Gate)

```text
- قبل إنشاء أي TASK جديدة أو تنفيذها أو تفويضها لعميل فرعي،
  يجب على Tera مراجعة PROJECT_STATE.md.
- إذا كان PROJECT_STATE.md يشير إلى PROJECT_RULES.md أو أي ملف
  قواعد معتمد آخر، يجب قراءة القواعد المرتبطة بالمهمة وتطبيقها،
  وعدم الاعتماد فقط على ذاكرة المحادثة.
- عند بداية كل TASK، يجب أن يحتوي ملف المهمة على قسم Context Check
  يوضح الملفات المرجعية التي راجعها Tera قبل التنفيذ، مثل:
  - PROJECT_STATE.md
  - PROJECT_RULES.md
  - TASK_REGISTRY.md
  - الملفات الفنية المرتبطة بالمهمة
```

### 1.4 قواعد إدارة قاعدة البيانات (Database Migrations)

```text
- في مهام قاعدة البيانات الرسمية، لا يستخدم Tera الأمر
  npx prisma db push إلا إذا تمت الموافقة عليه صراحة.
- المسار المعتمد لإنشاء الجداول هو Prisma Migrations باستخدام
  npx prisma migrate dev مع اسم migration واضح.
- عند تنفيذ أي مهمة تغير قاعدة البيانات، يجب أن ينتج عنها أثر قابل
  للتتبع داخل prisma/migrations ما لم يتم اعتماد استثناء صريح.
```

## 4. قاعدة الأولوية

ترتيب الأولوية:

1. قواعد السلامة والأمان.
2. تعليمات النظام وملفات `tera-system/`.
3. `PROJECT_RULES.md`.
4. ملفات التحضير المعتمدة داخل `project-preparation/`.
5. ملاحظات المحادثة غير المحفوظة.

لا يجوز استخدام هذا الملف لتجاوز قيود الأمان أو توسيع النطاق دون اعتماد Tera وصاحب المشروع.
