# Design Source

This folder stores raw or user-provided visual design sources for a project.

Use this folder only when Tera has decided that the project has a specific UI design source.

Accepted content examples:

- `theme.css`
- `tokens.css`
- `getdesign-style.md`
- screenshots or reference notes
- component style notes
- brand or color guidelines

Rules:

- Do not store secrets or credentials here.
- Do not treat this folder as implementation code.
- Raw design sources should be summarized into `project-preparation/28_UI_UX_GUIDELINES.md` before UI implementation.
- Engineering agents must follow the approved summary and must not invent conflicting UI styles.
- If multiple design sources conflict, Tera must resolve the conflict before implementation.
