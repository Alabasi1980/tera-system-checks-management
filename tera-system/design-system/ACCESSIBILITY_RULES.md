# Accessibility Rules

## Purpose

Set baseline accessibility requirements for UI tasks.

## Required Baseline

- Text contrast must be readable on all backgrounds.
- Focus states must be visible for keyboard navigation.
- Buttons and links must have clear labels.
- Form fields must have labels and error states.
- Color must not be the only indicator of status.
- Click/tap targets should be practical on mobile.
- Loading, empty, success, and error states must be planned for important workflows.

## UI Acceptance Checks

For each UI task:

```text
Contrast checked: Yes / No / N/A
Focus states present: Yes / No / N/A
Responsive behavior defined: Yes / No / N/A
RTL/LTR respected: Yes / No / N/A
Error/empty/loading states defined: Yes / No / N/A
```

## Rule

If accessibility behavior is missing from the design source, record a Design Gap instead of guessing.
