# Component Library Schema

## Purpose

Define how component style rules should be recorded before implementation.

## Required Component Rule Format

```text
Component:
Purpose:
Variants:
States:
Spacing:
Typography:
Color tokens:
Interaction:
Accessibility:
Forbidden:
```

## Core Components

- Button
- Input
- Select
- Textarea
- Checkbox / Radio
- Card
- Table
- Sidebar
- Topbar
- Tabs
- Modal / Drawer
- Alert / Toast
- Badge / Tag
- Pagination
- Empty State
- Loading State

## Rules

- Do not create a new component style when an approved pattern exists.
- Reuse existing implemented components before adding variants.
- Any new variant must map to a task acceptance criterion.
- UI tasks must list the components affected.
