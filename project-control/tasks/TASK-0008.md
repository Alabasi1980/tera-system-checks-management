## 4. المخرجات المطلوبة

### 4.1 EngineeringAgent — `app/parties/actions.ts`

```text
ملف: checks-management/app/parties/actions.ts
مالك: EngineeringAgent
النمط: مطابق تمامًا لـ banks/actions.ts مع تعديل حسب Party model
```

المطلوب:

1. `'use server'`
2. import: `prisma`, `revalidatePath`, `requireAdmin`
3. أنواع TypeScript: `PartyItem`, `PartyFormData`
4. `listParties()` — `requireAdmin()` → `findMany({ orderBy: { name: 'asc' }, include: { _count: { select: { checks: true } } } })`
5. `createParty(data)` — `requireAdmin()` → `prisma.party.create({ data: { name, phone, notes } })` + `revalidatePath`
6. `updateParty(id, data)` — `requireAdmin()` → `prisma.party.update({ where: { id }, data: { name, phone, notes } })` + `revalidatePath`
7. `deleteParty(id)` — `requireAdmin()` → التحقق من `checkCount` باستخدام `prisma.check.count({ where: { partyId: id } })` → منع الحذف إذا `> 0` + رسالة منع
8. رسائل الخطأ بالعربية (بنفس نمط Banks)

**ملاحظات:**
- لا يوجد فحص unique على `name` (على عكس Banks)
- `phone` اختياري
- `notes` اختياري

### 4.2 FrontendAgent — `app/parties/page.tsx`

```text
ملف: checks-management/app/parties/page.tsx
مالك: FrontendAgent
النمط: مطابق تمامًا لـ banks/page.tsx مع تعديل حسب Party model و S04 spec
```

المطلوب:

1. `'use client'`
2. استيراد `listParties`, `createParty`, `updateParty`, `deleteParty` و types من `./actions`
3. نفس هيكل Banks page:
   - Header: "إدارة الجهات" + زر "إضافة جهة جديدة"
   - جدول: اسم الجهة، الهاتف، عدد الشيكات، تاريخ الإضافة، الإجراءات (تعديل، حذف)
   - Modal: إضافة/تعديل مع حقول: اسم الجهة (مطلوب*، max 100), الهاتف (اختياري، max 50), ملاحظات (اختياري، max 500, textarea)
   - Toast للنجاح/الخطأ
   - تأكيد الحذف
   - حالة فارغة: "لا توجد جهات مسجلة بعد."
4. رسائل بالعربية:
   - نجاح: "تم إضافة الجهة بنجاح" / "تم تعديل الجهة بنجاح" / "تم حذف الجهة بنجاح"
   - خطأ الحذف: "لا يمكن حذف هذه الجهة لأنها مرتبطة بشيكات موجودة. قم بحذف أو إعادة ربط الشيكات المرتبطة أولاً."
5. نفس الألوان والستايل المطبق في Banks (حسب `28_UI_UX_GUIDELINES.md`)
6. نفس hover effects, focus border, disabled state
