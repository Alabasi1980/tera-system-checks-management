## Exact Instructions

### Step 1: إنشاء `.env` مع بيانات الاتصال الحقيقية

أنشئ ملف `.env` في جذر المشروع (`checks-management/.env`) باستخدام قيمة الاتصال المحلية التي قدمها المستخدم، ولا توثق السر هنا. الصيغة المرجعية فقط:

```
DATABASE_URL="postgresql://postgres:[REDACTED]@localhost:5432/checks_management"
```

ملاحظة: إذا احتوت كلمة المرور على رموز خاصة مثل `@` فيجب URL-encoding داخل القيمة المحلية الفعلية.

### Step 2: إنشاء قاعدة البيانات

قبل تشغيل Prisma Migration، تأكد من وجود قاعدة البيانات. استخدم إحدى الطريقتين حسب المتاح:

**الطريقة أ (يفضل):** باستخدام `createdb`:
```bash
createdb -h localhost -U postgres checks_management
```
سيطلب كلمة المرور: أدخل كلمة المرور المحلية التي قدمها المستخدم

إذا لم يكن `createdb` متاحًا في PATH، جرّب المسار الكامل:
```bash
& "C:\Program Files\PostgreSQL\16\bin\createdb.exe" -h localhost -U postgres checks_management
```

**الطريقة ب (بديل):** باستخدام `psql`:
```powershell
$env:PGPASSWORD = "[REDACTED]"
psql -h localhost -U postgres -c "CREATE DATABASE checks_management;"
```

**الطريقة ج (بديل PowerShell):** إذا لم يتوفر `psql` أو `createdb`، استخدم PowerShell مع .NET:
```powershell
$conn = New-Object System.Data.Odbc.OdbcConnection
$conn.ConnectionString = "Driver={PostgreSQL Unicode};Server=localhost;Port=5432;Database=postgres;Uid=postgres;Pwd=[REDACTED];"
$conn.Open()
$cmd = $conn.CreateCommand()
$cmd.CommandText = "CREATE DATABASE checks_management"
$cmd.ExecuteNonQuery()
$conn.Close()
```

إذا كانت قاعدة البيانات موجودة مسبقًا، تجاهل الخطأ واستمر.

### Step 3: تشغيل Prisma Migration

بعد إنشاء قاعدة البيانات، شغّل أمر migration:

```bash
npx prisma migrate dev --name init
```

يجب أن:
- ينشئ مجلد `prisma/migrations/` مع ملف migration جديد
- ينفذ الـ SQL لإنشاء الجداول الأربعة (users, banks, parties, checks)
- ينتهي بنجاح (✔)

### Step 4: التحقق

- تأكد من وجود مجلد `prisma/migrations/` مع migration واحد على الأقل.
- تأكد من أن `npx prisma generate` يعمل (اختياري — قد يكون تم تلقائيًا مع migrate dev).
