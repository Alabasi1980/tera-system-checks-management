## 2. Objective

Fix `ISSUE-0006` by adding server-side validation and safe normalization to Banks and Parties Server Actions before building Checks S02.

---

## 3. Scope

### In Scope

- Add validation helpers inside `banks/actions.ts` and `parties/actions.ts`.
- Validate required `name` server-side.
- Trim string fields before database writes.
- Enforce max lengths server-side:
  - Bank: `name` 100, `branch` 100, `accountNumber` 50, `notes` 500.
  - Party: `name` 100, `phone` 50, `notes` 500.
- Keep existing Arabic error messages style.
- Keep `requireAdmin()` at the top of every action.
- Keep existing bank duplicate-name behavior, but use normalized trimmed name.
- Keep Parties without duplicate-name check.

### Out of Scope

- No UI changes.
- No schema changes.
- No migrations.
- No API routes.
- No shared validation library unless strictly necessary.
- No changes to Checks S02 or Users S05.
