## 1. مسار العمل الرئيسي: إدارة الوصفات (Admin)

```
1. Admin ← تسجيل الدخول إلى لوحة التحكم (/admin/login)
       │
2. ← Dashboard الرئيسي ← إحصائيات سريعة
       │
3. ← إدارة التصنيفات (إن وجدت — تأكد من وجود تصنيفات للوصفة)
       │
4. ← إدارة المكونات (إن وجدت — تأكد من وجود مكونات للوصفة)
       │
5. ← إضافة وصفة جديدة (/admin/recipes/new)
       │
       ├── 5.1 اختيار التصنيف (قائمة منسدلة)
       ├── 5.2 إضافة المكونات:
       │       └── اختيار مكون من القائمة + تحديد الكمية + وحدة القياس
       ├── 5.3 إضافة خطوات التحضير (نص منسق، قائمة مرقمة)
       ├── 5.4 رفع الصور (اختياري — صور متعددة)
       ├── 5.5 إضافة رابط فيديو YouTube (اختياري)
       ├── 5.6 إضافة وقت التحضير والطهي ⏱️ (1B)
       │
6. ← حفظ الوصفة
       │
7. ← الوصفة تظهر فوراً في الواجهة العامة
```

## 2. مسار العمل: تصفح الزوار

```
1. زائر ← فتح الموقع (/)
       │
       ├── ← الصفحة الرئيسية (أحدث الوصفات، مصنفة)
       ├── ← صفحة التصنيفات (/categories)
       │       └── اختيار تصنيف ← عرض وصفاته
       ├── ← بحث (/search?q=...)
       └──
2. ← فتح وصفة (/recipes/[id])
       │
       ├── مشاهدة المقدار (مع إمكانية التكبير/التصغير 📐 1B)
       ├── مشاهدة خطوات التحضير
       ├── مشاهدة معرض الصور
       ├── مشاهدة فيديو YouTube (إن وجد)
       ├── تحميل PDF (1B)
       ├── حفظ في المفضلة ❤️ (1B)
       └── مشاركة الوصفة
```

## 3. مسار العمل: التعليقات

```
— كتابة تعليق —
1. زائر ← فتح وصفة
2. التمرير إلى قسم التعليقات
3. إدخال الاسم (حقل إجباري)
4. كتابة التعليق (نص)
5. إرسال
6. ← التعليق يُحفظ بحالة "قيد المراجعة" (pending)

— مراجعة التعليق —
7. Admin ← لوحة التحكم ← إدارة التعليقات
8. عرض التعليقات المعلقة
9. اختيار:
       ├── ✅ موافقة → التعليق يظهر في الواجهة العامة
       └── ❌ حذف → التعليق يُحذف نهائياً
```

---

## 4. مسار العمل: قائمة المشتريات الذكية 🛒 (1B)

```
1. زائر يفتح وصفة
2. يضغط "أضف المكونات إلى قائمة المشتريات"
3. يتم تجميع مكونات الوصفة في قائمة مؤقتة
4. يمكن إضافة مكونات من وصفات متعددة
5. عرض القائمة مجمعة (المكون نفسه يظهر مرة واحدة بالكمية الإجمالية)
6. طباعة القائمة أو نسخها
7. القائمة مخزنة محلياً (LocalStorage)
```

---

## 5. مسار العمل: مقياس الوصفات الديناميكي 📐 (1B)

```
1. زائر يفتح وصفة
2. في قسم "المقادير" ← أداة تحجيم
3. إدخال العدد المطلوب من الأشخاص (مثلاً: تكفي 4 بدل 2)
4. النظام يحسب النسبة: العدد المطلوب ÷ العدد الأصلي
5. تحديث جميع الكميات في الواجهة (بدون تغيير في قاعدة البيانات)
6. إعادة الحساب عند تغيير العدد
```

---

## 6. مسار العمل: المفضلة ❤️ (1B)

```
1. زائر يفتح وصفة
2. يضغط على أيقونة القلب ❤️
3. الوصفة تُحفظ في قائمة المفضلة (LocalStorage)
4. يمكن عرض قائمة المفضلة من أيقونة في الشريط العلوي
5. يمكن إزالة وصفة من المفضلة بالضغط مرة أخرى
```

---

## 7. مسار العمل: وقت التحضير والطهي ⏱️ (1B)

```
— الإضافة —
1. Admin عند إنشاء/تعديل وصفة
2. إدخال وقت التحضير (بالدقائق)
3. إدخال وقت الطهي (بالدقائق)
4. حفظ

— العرض والفلترة —
5. عرض الوقت في بطاقة الوصفة وصفحة التفاصيل
6. إمكانية البحث/الفلترة بالوقت الإجمالي (<= 15 د، <= 30 د، <= 60 د، > 60 د)
```

---

## 8. مسار العمل: تحميل PDF (1B)

```
1. زائر يفتح وصفة
2. يضغط "تحميل PDF"
3. النظام يُنشئ ملف PDF منسق:
       ├── عنوان الوصفة
       ├── صورة الوصفة
       ├── قائمة المكونات مع الكميات
       ├── خطوات التحضير
       ├── وقت التحضير والطهي (إن وجد)
       └── ملاحظات
4. تنزيل الملف للمستخدم
```
