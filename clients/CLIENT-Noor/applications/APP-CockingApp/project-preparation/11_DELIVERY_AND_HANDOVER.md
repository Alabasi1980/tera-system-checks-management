## 1. مخرجات التسليم (Deliverables)

| المخرج | الوصف | التنسيق |
|--------|-------|---------|
| **شفرة المصدر** | كود التطبيق الكامل | مستودع Git |

## 2. مراحل التسليم

### 2.1 قبل التسليم

- [ ] جميع TASK-IDs في حالة Closed
- [ ] الاختبار اليدوي مكتمل
- [ ] `PROJECT_STATE.md` محدث
- [ ] `RELEASE_NOTES.md` يحتوي على ملخص الإصدار
- [ ] `ISSUES_AND_GAPS.md` لا يحتوي على Critical items

### 2.2 التسليم للعميل

1. **تنظيف الكود**: إزالة التعليقات غير الضرورية، logs
2. **بناء الإنتاج**: `npm run build` يعمل بدون أخطاء
3. **تصدير قاعدة البيانات**: `npx prisma migrate deploy` جاهز
4. **إعداد ملف `.env`**: بقيم الإنتاج
5. **تقديم العرض**: شرح التطبيق للعميل (نور)

### 2.3 بعد التسليم

- فترة ضمان: (تحدد لاحقاً)
- الدعم: حسب الاتفاق
- التقارير الإضافية: حسب الطلب

## 3. التوثيق للعميل

### 3.1 دليل الاستخدام (لـ نور — Admin)

يجب أن يشرح:

1. **تسجيل الدخول**: كيف تدخل لوحة التحكم
2. **إضافة وصفة**: خطوة بخطوة — من البداية للنشر
3. **إدارة التصنيفات**: إضافة، تعديل، حذف، ترتيب
4. **إدارة المكونات**: إضافة مكون جديد، اختيار وحدة القياس
5. **إدارة التعليقات**: موافقة، حذف
6. **تحميل PDF**: كيف تحمل الزوار PDF
7. **الصور**: ماذا تفعل إذا لم تظهر الصور

### 3.2 دليل النشر (لـ مشرف السيرفر)

1. متطلبات السيرفر
2. خطوات التثبيت
3. إعداد قاعدة البيانات
4. إعداد `.env`
5. التشغيل والإيقاف
6. النسخ الاحتياطي والاستعادة
