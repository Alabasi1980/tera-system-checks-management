## 1. قواعد التعليقات

| # | القاعدة | التفاصيل |
|---|---------|----------|
| BR-01 | التعليقات عامة بدون حساب | أي زائر يمكنه التعليق بدون تسجيل دخول |
| BR-02 | حقل الاسم إجباري | يجب على المعلق إدخال اسم (نص عادي، max 100 حرف) |
| BR-03 | الموافقة المسبقة | كل تعليق جديد بحالة `isApproved = false` ولا يظهر للزوار حتى يوافق Admin |
| BR-04 | إشعار التعليقات | لا حاجة لإشعار — Admin يرى التعليقات المعلقة في Dashboard |
| BR-05 | منع SPAM مبدئي | منع الروابط في التعليقات (اختياري) + حد أقصى للحرف (1000) |
| BR-06 | حذف التعليق | Admin فقط يمكنه حذف تعليق (حذف نهائي من قاعدة البيانات) |
| BR-07 | لا تعديل | المعلق لا يمكنه تعديل أو حذف تعليقه بعد الإرسال |

---

## 2. قواعد المكونات

| # | القاعدة | التفاصيل |
|---|---------|----------|
| BR-08 | المكونات ككيانات منفصلة | لا نصوص حرة — كل مكون مسجل ككيان في قاعدة البيانات |
| BR-09 | اسم المكون فريد | لا يمكن إضافة مكون بنفس الاسم (case-insensitive) |
| BR-10 | وحدة قياس افتراضية | لكل مكون وحدة قياس افتراضية (مثلاً: دقيق ← GRAM) |
| BR-11 | تغيير الوحدة في الوصفة | يمكن تغيير وحدة المكون في وصفة معينة (مثلاً: دقيق بالكوب بدل الغرام) |
| BR-12 | حذف المكون | لا يمكن حذف مكون مستخدم في وصفات — يمنع أو تنبيه |
| BR-13 | الكمية صفر أو سالب | لا يُسمح — التحقق من أن الكمية > 0 |

---

## 3. قواعد التصنيفات

| # | القاعدة | التفاصيل |
|---|---------|----------|
| BR-14 | اسم التصنيف فريد | لا يمكن إضافة تصنيف بنفس الاسم |
| BR-15 | حذف التصنيف | إذا كان التصنيف يحتوي وصفات، يمنع الحذف أو يُطلب إعادة تعيين الوصفات |
| BR-16 | ترتيب التصنيفات | Admin يمكنه ترتيب التصنيفات (sortOrder) |

---

## 4. قواعد الوصفات

| # | القاعدة | التفاصيل |
|---|---------|----------|
| BR-17 | عنوان الوصفة مطلوب | حقل title إجباري (max 200 حرف) |
| BR-18 | تصنيف الوصفة مطلوب | يجب اختيار تصنيف واحد على الأقل |
| BR-19 | مكون واحد على الأقل | لا يمكن حفظ وصفة بدون مكونات |
| BR-20 | خطوة واحدة على الأقل | لا يمكن حفظ وصفة بدون خطوات تحضير |
| BR-21 | حالة النشر | الوصفة تنشأ كمسودة (isPublished = false) — Admin ينشرها يدوياً |
| BR-22 | الصور اختيارية | يمكن إضافة وصفة بدون صور |
| BR-23 | رابط YouTube اختياري | وإذا وُجد، يجب أن يكون رابط YouTube صحيحاً |

| # | القاعدة | التفاصيل |
|---|---------|----------|

## 6. قواعد القياس الديناميكي 📐 (1B)

| # | القاعدة | التفاصيل |
|---|---------|----------|
| BR-27 | العدد الأصلي | يتم تخزين العدد الأصلي للأشخاص في حقل `servings` |
| BR-28 | الحساب | عند تغيير العدد: `الكمية الجديدة = الكمية الأصلية × (العدد الجديد ÷ العدد الأصلي)` |
| BR-29 | لا تعديل دائم | المقياس الديناميكي مؤقت — لا يُحفظ في قاعدة البيانات |
| BR-30 | إعادة التعيين | زر "إعادة تعيين" يعيد الكميات إلى الأصلية |
