## 1. الأدوار

| المعرف | الدور | العدد | الوصف |
|--------|-------|-------|-------|
| **ADMIN** | Admin (نور) | 1 | مدير التطبيق — صلاحية كاملة على لوحة التحكم |

> **ملاحظة**: لا يوجد دور "مستخدم مسجل" — الزوار لا يسجلون دخولاً.
> **ملاحظة**: لا يوجد نظام أدوار متعددة — Admin واحد فقط حسب طلب العميل.

## 3. تدفق المصادقة (Admin)

```
1. Admin يفتح /admin/login
2. إدخال اسم المستخدم وكلمة المرور
3. التحقق → إنشاء session/token
4. توجيه إلى /admin/dashboard
5. الجلسة صالحة لمدة محددة → انتهاء الجلسة → طلب إعادة تسجيل الدخول
```

> **طريقة Auth**: جلسات بسيطة (NextAuth أو JWT) — حسب ما يناسب on-premise deployment.

---

## 4. صلاحيات الزوار (بدون حساب)

الزوار ليسوا مجهولين تماماً — لكن لا يوجد لديهم حساب:
- يمكنهم التعليق (يدخلون اسماً فقط)
- يمكنهم حفظ المفضلة (في المتصفح — LocalStorage)
- لا يمكنهم تعديل أو حذف تعليقاتهم (تحتاج موافقة Admin)

---

## 5. قيود الأمان

| القيد | التفاصيل |
|-------|---------|
| حماية لوحة التحكم | مسار /admin محمي — إعادة توجيه للزوار غير المصرح لهم |
| كلمة المرور | يُفضل تخزينها بشكل آمن (bcrypt) |
| الجلسة | timeout مناسب — انتهاء الجلسة بعد فترة عدم نشاط |
| API Routes | حماية جميع routes تحت /admin/api |
| رفع الصور | التحقق من نوع الملف وحجمه |
| التعليقات | فلترة ضد SPAM (مؤقتة) + موافقة Admin |
